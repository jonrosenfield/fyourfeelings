## 3. NEW ARCHITECTURE

### Stack Decision: Vanilla HTML/CSS/JS (Enhanced)

**Rationale:** No framework switch needed. The existing vanilla approach works well for this
static site. We're enhancing it with:
- A **client-side RSS parser** using `fetch()` + `DOMParser` (no build step needed)
- **Self-hosted fonts** (Space Mono + Inter from Google Fonts downloaded locally)
- **CSS custom properties** for the new Tactile Rebellion palette
- **Intersection Observer API** for scroll-triggered animations
- **Web Animations API** for kinetic typography
- **MediaRecorder API** (already used, kept for submit-story)
- **Netlify Forms** (kept for form submissions — zero backend needed)

**No npm, no build tools, no framework.** Deployable as static files to Netlify.

### Typography
- **Primary (headers/UI):** Space Mono — self-hosted WOFF2
- **Body:** Inter — self-hosted WOFF2
- `font-display: swap` on all font-face declarations

### Key Aesthetic Elements
- CSS `@keyframes` grain flicker overlay using SVG filter
- Scroll-triggered kinetic hero text (scale + slight skew via IntersectionObserver)
- Hand-drawn SVG dividers between sections (intentionally rough)
- Glitch effect on hero headline using `clip-path` animation
- Episode cards with subtle depth/tilt on hover (CSS perspective transform)
- No light mode in v2 — dark only (brand decision for Tactile Rebellion aesthetic)

## 6. PAGE ARCHITECTURE

### index.html — Scrollytelling Homepage

Sections (in scroll order):
1. **Hero** — Kinetic headline, glitch animation, latest episode auto-loaded from RSS
2. **Who We Are** — Manifesto copy (3 hosts, raw tone)
3. **Episodes Feed** — Card grid from RSS (latest 6 shown, "Load More" for rest)
4. **Clips / Shorts** — YouTube Shorts floating card layout
5. **Community CTA** — Spotify follow button + email capture

Schema: `PodcastSeries` (JSON-LD in `<head>`)
FAQ Schema: 4 questions about the show

### episode.html — Dynamic Episode Template

URL Pattern: `/episode.html?ep=18` or `/episodes/18` (via Netlify redirect)
Rendered client-side from RSS data (parsed and cached in sessionStorage)

Includes:
- Episode title + number + date
- Embedded `<audio>` player with custom controls
- Full show notes (HTML from RSS description)
- Social share: Spotify, Apple Podcasts, copy link
- Schema: `PodcastEpisode` JSON-LD

### submit-story.html — Story Submission

Methods:
1. **Record audio** (MediaRecorder API → WebM)
2. **Upload audio** (MP3, WAV, M4A, max 10MB)
3. **Submit text** (textarea)

Backend: Netlify Forms (zero-config, just `data-netlify="true"`)

## 7. PERFORMANCE TARGETS

| Metric | Target |
|--------|--------|
| LCP | < 2.5s |
| FID/INP | < 100ms |
| CLS | < 0.1 |
| Lighthouse Score | ≥ 85 across all categories |

**Strategies:**
- Fonts: self-hosted WOFF2, `font-display: swap`
- Images: `loading="lazy"` below fold, `fetchpriority="high"` on hero image
- YouTube iframes: facade pattern (thumbnail + click-to-load)
- RSS: fetched client-side on DOMContentLoaded, cached in sessionStorage (5min TTL)
- No render-blocking scripts (`defer` on all JS)
- Critical CSS inlined in `<head>` for above-fold content
